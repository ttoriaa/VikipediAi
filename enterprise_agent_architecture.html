Enterprise Agent Architecture

企业级四类 Agent 架构

目标是把 AI 能力从“会回答”升级到“会办事”：由上层负责识别与编排，由中层负责业务执行， 由底层负责受控动作执行，形成可观测、可治理、可规模化的智能体体系。

总入口智能体 Super Agent

识别 + 分发 + 编排触发，而不是替代领域执行。

  • 统一用户入口与对话体验，收集上下文。
  • 识别用户意图并分发至合适下游。
  • 触发后续流程，但不直接承担深业务决策。

调度智能体 Orchestrator Agent

定位：多智能体系统中的“项目经理/调度中心”。

  • 将复杂目标拆解为可执行子任务。
  • 规划执行顺序、并行关系与依赖。
  • 处理异常、回退、重试和人工接管。

领域智能体 Domain Agent

定位：对具体业务结果负责的“业务执行主体”。

  • 理解专业语境与业务规则，完成端到端任务。
  • 直接对结果 KPI 负责，如转化率、AHT、FCR 等。
  • 协同多个系统执行，形成业务闭环。

工具与连接器 Tool / Connector

定位：把企业内外系统能力封装成“受管控动作”的执行层。

  • 封装受控动作：查询、创建、更新、提交。
  • 统一权限控制、审计日志与高风险拦截。
  • 只执行，不决策，不规划。

协同链路

Super 识别与分发
→
Orchestrator 拆解与调度
→
Domain 业务执行与结果负责
→
Tool/Connector 受控动作执行

落地检查点

  1. 是否有统一入口，避免多个孤立 AI 系统并存。
  2. 是否有独立调度层来管理复杂流程与异常回退。
  3. 每个领域智能体是否绑定清晰业务 KPI 和责任人。
  4. 高风险动作是否全部下沉到 Tool/Connector 做统一治理。
  5. 四层日志是否可追溯到同一任务链路。
