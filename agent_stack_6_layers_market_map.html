智能体六层架构与全市场产品映射

统一口径：用户连接 -> 智能体 -> Skill -> Harness -> 模型服务 -> Infra。该页面用于解释每层职责边界，并给出覆盖全球与国内主流产品的映射参考。

一、六层总体视图

一个可落地的企业级 Agent 系统，通常是跨层组合而不是单层产品。下面先看统一流程，再看逐层解释。

L1 用户连接
L2 智能体
L3 Skill
L4 Harness
L5 模型服务
L6 Infra

二、六层详细解释

L1 用户连接层（User Connectivity）

核心职责
连接用户与系统，管理身份、会话和入口渠道。
关键能力
多渠道接入、SSO、RBAC、会话上下文注入、终端适配。
输入
用户请求、身份信息、组织权限、历史会话。
输出
标准化任务请求 + 可信上下文。
指标
请求成功率、登录成功率、会话连续性。
典型风险
身份漂移、权限越权、上下文污染。

L2 智能体层（Agent Layer）

核心职责
把意图转为计划，决定“做什么、按什么顺序做”。
关键能力
任务拆解、规划、反思重试、多智能体协作、状态机。
输入
用户任务 + 业务上下文 + 策略约束。
输出
可执行 action plan。
指标
任务完成率、步骤正确率、平均回合数。
典型风险
幻觉计划、循环调用、路径漂移。

L3 Skill 层（Capability Layer）

核心职责
封装可复用业务能力，提供稳定调用接口。
关键能力
检索、报表、审批、代码处理、行业规则、RPA 连接。
输入
Agent 下发参数。
输出
结构化业务结果。
指标
调用成功率、单次耗时、复用率。
典型风险
接口碎片化、版本不兼容、权限不一致。

L4 Harness 层（Runtime & Governance）

核心职责
让计划可控执行：安全、稳定、可观测、可审计。
关键能力
执行编排、工具网关、沙箱、策略门禁、审计日志、回放评测、成本治理。
输入
Agent 计划 + Skill 工具列表 + 组织策略。
输出
受控执行结果 + 全链路日志。
指标
完成率、恢复率、误拦截率、单位任务成本。
典型风险
策略过严/过松、执行抖动、审计缺失。

L5 模型服务层（Model Service）

核心职责
提供推理能力并做多模型路由。
关键能力
模型网关、路由/Fallback、上下文管理、推理参数策略、函数调用适配。
输入
Prompt、上下文、工具约束。
输出
模型结果、函数调用指令、置信信号。
指标
质量、延迟、token 成本、可用性。
典型风险
单厂商锁定、成本失控、能力漂移。

L6 Infra 层（Infrastructure）

核心职责
提供算力、存储、网络与安全底座。
关键能力
Kubernetes、GPU 调度、对象存储、向量库、密钥管理、容灾、FinOps。
输入
运行负载、数据与安全策略。
输出
可扩展、可靠、可控成本的运行环境。
指标
可用性、吞吐、P95 延迟、单位算力成本。
典型风险
资源热点、网络瓶颈、合规与跨境风险。

三、边界与协作（避免混层）

最常见误区：把 Agent、Harness、模型服务混成一层。下面给出职责边界。

相邻层 上层回答的问题 下层回答的问题 边界关键词
L2 Agent vs L4 Harness 要做什么（Plan） 如何安全稳定做完（Execute & Govern） 计划决策 vs 运行治理
L3 Skill vs L4 Harness 能力本体（Business Capability） 能力调用控制（Policy, Retry, Audit） 业务函数 vs 运行控制
L4 Harness vs L5 Model Service 执行与治理 模型推理能力与路由 控制平面 vs 推理平面
